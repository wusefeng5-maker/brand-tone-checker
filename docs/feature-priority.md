# Feature Priority

## Priority Model

Features are ranked by:

- User value
- Development cost
- Competitive moat
- AI replaceability
- Fit with AI Brand QA Platform positioning

Prompt optimization is not a P0 priority. Prompt quality matters, but it is not the product moat.

## P0

### 1. Brand Brain

The long-term brand memory that combines profile data, examples, rules, channel preferences, feedback, and history.

Why:

- Reduces setup friction.
- Creates user-owned assets.
- Makes future checks more accurate.
- Cannot be easily replaced by a one-off chatbot.

### 2. Evidence Engine

Turns QA output into explainable findings:

- Sentence
- Issue
- Rule
- Source
- Severity
- Suggested revision

Why:

- Builds trust.
- Moves product from AI opinion to QA evidence.
- Helps agencies explain revisions to clients.

### 3. Pass / Needs Revision / Off Brand

A clear QA decision.

Why:

- Users do not only want a score.
- The real question is whether content can be published.
- Supports approval workflow later.

### 4. Rule Engine

A non-AI and AI-assisted standard system.

Examples:

- Forbidden words
- Preferred phrases
- Required messages
- Channel constraints
- Tone dimensions
- Legal risk expressions

Why:

- Rules are repeatable and explainable.
- Rules reduce dependence on model behavior.
- Rules become brand assets.

### 5. High Score Library

Approved examples that represent the brand.

Why:

- Makes the system improve over time.
- Helps new team members learn brand voice.
- Gives future checks concrete references.

### 6. Feedback Learning

The user can correct the system:

- Useful
- Inaccurate
- Save as rule
- Save as example
- Add to forbidden expression

Why:

- Creates a learning loop.
- Converts daily usage into brand knowledge.
- Builds switching cost.

### 7. Context

Each check should know:

- Platform
- Audience
- Goal

Why:

- Brand fit depends on context.
- A Xiaohongshu note, ecommerce detail page, and investor email should not use the same criteria.

## P1

### 1. Batch QA

Check multiple pieces of content at once.

Why:

- High agency value.
- Saves real time.
- Harder to replace with manual ChatGPT workflows.

### 2. Brand Workspace

A workspace for each brand or client.

Why:

- Supports agency ICP.
- Organizes Brand Brain, rules, history, and reports.

### 3. Shareable QA Report

Client-ready report explaining what changed and why.

Why:

- Directly supports agency delivery.
- Makes product output useful outside the app.

### 4. Channel Profiles

Channel-specific standards.

Why:

- Increases specificity.
- Strong fit for social, ecommerce, and cross-border workflows.

### 5. Basic Billing and Usage Limits

Free, Pro, and Agency plans.

Why:

- Required for commercial validation.
- Enables pricing experiments.

## P2

### 1. Team Collaboration

Members, comments, and simple roles.

Why:

- Important for retention.
- Should follow proof that teams use the QA flow.

### 2. Approval Workflow

Draft, checked, approved, rejected.

Why:

- Creates operational stickiness.
- Should remain simple at first.

### 3. Analytics

Common issues, average score, violated rules, quality trend.

Why:

- Useful for managers.
- Needs enough usage data to be meaningful.

### 4. Brand Versioning

Track changes to brand standards.

Why:

- Valuable as brands evolve.
- More important after rules and examples are active.

## Future

### 1. Browser Extension

Useful but expensive to maintain.

### 2. Google Docs / Notion / CMS Integrations

Important for workflow embedding, but only after core QA is proven.

### 3. Enterprise RBAC

Do not build before enterprise demand exists.

### 4. Client Portal

Useful for agencies later.

### 5. Auto Publishing

Not core to QA. Avoid until the QA workflow is deeply used.

### 6. Advanced Prompt Management

Should remain internal. Do not sell prompt management as the main product.
